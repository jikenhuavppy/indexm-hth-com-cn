pg模拟器 · 资料索引与使用说明

基于 H5 环境的功能整理 · 信息密度适中

对于需要在移动端或浏览器中快速体验 PostgreSQL 核心交互的用户，pg模拟器 提供了一个轻量的查询沙箱。它并非完整数据库实例，而是围绕 SQL 解析、表结构预览和基础 CRUD 模拟构建的资料索引型工具，适合学习验证、方案初筛或演示场景。

核心能力摘要

工具覆盖了 PostgreSQL 常见的数据类型、索引策略、连接查询与聚合写法。内部维护了一套预置数据集（订单、用户、商品），可通过模拟器快速检验语句逻辑，无需搭建本地环境。所有操作均在浏览器前端完成，不向远程服务器发送用户输入的 SQL 文本。

预置模式 三张关联表，含主键、外键与常用索引定义
语法高亮 编辑器支持 SQL 关键词着色，初步格式提示
结果快照 返回最多 200 行模拟数据，支持导出 CSV

适用场景建议

当你需要快速验证一个查询的结构是否正确、对比不同写法的执行计划（仅文本级模拟），或者向同事/学员演示 PostgreSQL 的基础行为时，pg模拟器 可以省去本地安装与配置的步骤。尤其适合以下情况：

  • 教学培训中实时展示 SELECT / JOIN / GROUP BY 效果
  • 快速检查索引字段选择与 WHERE 条件的匹配逻辑
  • 在没有网络数据库环境的设备上做离线功能演示

使用注意事项

由于运行在浏览器沙箱中，pg模拟器 不会持久化用户数据，也无法执行 DDL、存储过程或触发器等高级特性。模拟的查询结果基于静态快照，并非真实数据库响应，因此不适合用于生产数据校验或性能基准测试。建议将其作为概念验证或学习辅助工具。

⚡ 提示：每次刷新页面后，预置数据会重置为初始状态。如需保留当前查询或结果，可使用导出功能保存为本地文件。

常见疑问简述

Q：模拟器支持所有 PostgreSQL 函数吗？
A：仅覆盖最常用的聚合、字符串、日期与数学函数。完整列表可在工具内置的「函数参考」面板中查阅。
Q：是否可以导入自己的表结构？
A：当前版本不开放自定义 DDL，但预置模式覆盖了多数基础场景。后续版本规划中已列入表结构编辑功能。
Q：模拟数据量级有多大？
A：每张表约 30–80 行记录，足以演示查询行为，但无法模拟大数据量下的索引效果。

如果你希望进一步了解 PostgreSQL 在实际项目中的配置与调优，或需要一个更接近真实环境的测试平台，推荐通过 pg模拟器 作为起点，再过渡到本地安装的数据库实例。工具本身保持轻量、无追踪，所有处理均在客户端完成。
